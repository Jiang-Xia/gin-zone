# 项目技术栈与业务模块分析

## 1. 总体架构

本项目是一个“多端一体”的前后端分离练手项目，包含三部分：

- `client`：uni-app 前端（Vue3 + uni-app 多端页面）
- `server`：Go + Gin 后端 API（含 WebSocket 实时聊天）
- `admin`：后台管理系统（React18 + TypeScript）

仓库根目录：`D:/study/myGithub2/gin-zone`

## 2. 前端（client）技术栈

### 2.1 框架与构建

- uni-app：`client/App.vue`、`client/pages.json`
- Vue 3 单文件组件：`*.vue`
- 构建与运行：`client/vite.config.js` + `@dcloudio/vite-plugin-uni`
- 跨端目标：H5 / APP / 小程序（通过 `#ifdef` 分支）

### 2.2 UI 组件与工具

- 组件库：`@tdesign/uniapp`（`package.json` 依赖中可见）
- 自定义组件：`client/common/components`、以及 `client/uni_modules/*`
- 图表组件：`qiun-data-charts`（在 `packageB/pages/chart/chart.vue` 中使用）
- 分包业务组件：
  - 主包：`client/pages/*`（博客/动态/聊天/我的）
  - 分包 A：`client/packageA/pages/*`（登录/资料/聊天详情/博客列表详情/发动态/AI聊天等）
  - 分包 B：`client/packageB/pages/*`（收银台与支付、图表 demo 等）

### 2.3 状态/全局能力

- 全局状态：`client/main.js` 使用 Pinia
- 全局变量/能力：`client/App.vue` 中的 `globalData`
  - WebSocket 任务 `socketTask`
  - 用户信息 `userInfo`
  - 初始化聊天 WebSocket 的 `initChat`

### 2.4 网络层与鉴权/加密

- 请求封装：`client/common/request/api.js`
  - 默认携带 `Authorization: zoneToken`
  - 统一 `complete` 处理返回码与错误提示
- 可选报文加密：
  - 使用 `sm-crypto`（SM2/SM4）
  - 通过后端 `common/signIn` 获取 `sessionId/workKey`

## 3. 后端（server）技术栈

### 3.1 Web 框架与中间件

- Go 1.23：`server/go.mod`
- Gin：`server/router/router.go`（`gin.Default()`、路由分组）
- swagger：`github.com/swaggo/gin-swagger`（接口文档 `/api/v1/swagger/*any`）

### 3.2 数据与能力依赖

- ORM/数据库：`gorm.io/gorm` + `gorm.io/driver/mysql`
- 缓存：`github.com/go-redis/redis/v9`
- 日志：`logrus` + rotatelogs 等（`server/pkg/log`）
- 计划任务：`robfig/cron/v3`
- AI：
  - `github.com/sashabaranov/go-openai`（直接调用 OpenAI）

### 3.3 实时聊天

- WebSocket：`github.com/gorilla/websocket`
- 在 `server/app/controller/mobile/chat.go` 中：
  - `/api/v1/mobile/chat`：WebSocket 升级
  - 支持心跳 `heartbeat`、文本消息 `text`
  - 群聊按群成员列表逐个推送

## 4. 管理后台（admin）技术栈

- React：`react` / `react-dom`（18）
- TypeScript：`typescript`
- 路由：`react-router-dom` v6
- UI：
  - `antd`（Ant Design）
  - `@ant-design/pro-components`（ProComponents）
  - 典型入口在 `admin/src/pages/login/index.tsx`
- 状态管理：Redux（`@reduxjs/toolkit`、`redux-persist`）
- 路由模块拆分：`admin/src/routers/modules/*`

## 5. 业务模块划分（前端 + 后端对应）

> 说明：前端通过 `server/router/router.go` 的分组路径访问后端；部分业务（例如博客与支付交易）会走 `middleware.ReverseProxy()` 转发到外部服务。

### 5.1 用户与鉴权（Base）

前端页面/入口：

- `client/pages/my/index.vue`：展示用户信息；登录入口跳到 `packageA/pages/my/login.vue`
- `client/packageA/pages/my/*`：`login.vue`、`setting.vue`（资料编辑）

核心接口（后端）：

- `POST /api/v1/base/users/login`（账号密码登录）
- `POST /api/v1/base/users`（注册）
- `GET /api/v1/base/users/info`（获取当前用户信息）
- `PATCH /api/v1/base/users/:id`（更新资料）
- `POST /api/v1/base/users/password`（改密）
- `POST /api/v1/base/auth/wxlogin`（微信授权登录）

上传接口：

- `POST /api/v1/base/upload`（头像/发布动态文件上传）

### 5.2 博客模块（Blog，反向代理到外部）

前端页面：

- 主包：`client/pages/blog/index.vue`
- 分包 A：
  - `packageA/pages/blog/articles/articles.vue`（文章列表）
  - `packageA/pages/blog/detail/detail.vue`（文章详情，富文本渲染）

调用方式：

- `server/router/router.go` 中 `blog` 分组使用 `middleware.ReverseProxy()` 直接转发：
  - `/api/v1/blog/article/list`
  - `/api/v1/blog/article/info`
  - `/api/v1/blog/tag`
  - `/api/v1/blog/category`
  - `/api/v1/blog/comment/findAll`
  - `/api/v1/blog/resources/daily-img`

### 5.3 动态模块（Moment，后端本服务提供）

前端页面：

- 主包：`client/pages/moment/index.vue`
- 分包 A：`client/packageA/pages/moment/addMoment.vue`（发动态：文字 + 文件上传 + 定位）

核心接口（后端）：

- `GET /api/v1/mobile/moments`（动态列表）
- `POST /api/v1/mobile/moments`（发动态）
- `GET /api/v1/mobile/moments/UpdateMoment`（点赞/浏览：`t=like|view`）

### 5.4 聊天模块（Chat，WebSocket + REST）

前端页面：

- 主包：`client/pages/chat/index.vue`
  - 好友/群聊列表
  - WebSocket 消息推送更新最后一条消息与未读角标
- 分包 A：
  - `packageA/pages/chat/chatDetail/chatDetail.vue`（聊天详情：支持文本/图片/视频/语音 + 历史消息分页）
  - `packageA/pages/chat/addFriend.vue`（添加好友/加入群）
  - `packageA/pages/my/createGroup.vue`（创建群聊）

核心接口（后端）：

- WebSocket：
  - `GET /api/v1/mobile/chat?userId=...`
- REST：
  - `GET /api/v1/mobile/chat/friends`
  - `POST /api/v1/mobile/chat/friends`（添加好友/会话）
  - `DELETE /api/v1/mobile/chat/friends/:friendId`
  - `GET /api/v1/mobile/chat/groups`
  - `POST /api/v1/mobile/chat/groups`
  - `DELETE /api/v1/mobile/chat/groups/:groupId`
  - `GET /api/v1/mobile/chat/groupMembers`
  - `POST /api/v1/mobile/chat/groupMembers`
  - `DELETE /api/v1/mobile/chat/groupMembers/:groupId`
  - `POST /api/v1/mobile/chat/logs`（历史消息分页）
  - `POST /api/v1/mobile/chat/updateReadTime`（更新阅读时间）

### 5.5 AI 聊天（Third / ChatGPT）

前端页面：

- `client/packageA/pages/chat/chatAiDetail/chatAiDetail.vue`
  - 选择不同“场景”
  - 调用后端 AI 接口生成回复

核心接口（后端）：

- `POST /api/v1/third/chatGPT`（go-openai 直连 OpenAI）
- `POST /api/v1/third/chatGPTApi`（代理到外部接口 `chat.xingyijun.cn`）
- `GET /api/v1/third/gushici`（今日古诗词，redis 缓存 + cron 刷新）

### 5.6 收银台与支付（Business Pay，packageB，部分转发到外部 blog-server）

前端页面（分包 B）：

- `packageB/pages/business/pay/cashier/cashier.vue`：金额输入/备注、获取二维码信息、跳转支付页
- `packageB/pages/business/pay/transfer/transfer.vue`：微信/支付宝授权中转页（拿到 `code` 后重定向到 cashier）
- `packageB/pages/business/pay/all-pay/all-pay.vue`：选择支付方式、发起 `ApplyOrder`、调用 `PayOrder`，并处理跳转/小程序支付
- `packageB/pages/business/pay/host-pay/host-pay.vue`：本行卡支付（手机号验证码 `GetValidateCode`、银行卡查询 `QjbankCardQuery`）
- `packageB/pages/business/pay/status/status.vue`：支付结果展示（成功失败/处理中轮询或查询）
- `packageB/pages/business/webview/webview.vue`：手机号快捷登录 WebView（定位 + 确认协议弹窗）

后端入口（转发）：

- `server/router/router.go` 中 `blog` 分组对支付交易做了反向代理：
  - `POST /api/v1/blog/pay/trade/create`
  - `GET  /api/v1/blog/pay/trade/query`
  - `POST /api/v1/blog/pay/trade/refund`
  - `POST /api/v1/blog/pay/trade/close`
  - `POST /api/v1/blog/pay/openid`
  - `POST /api/v1/blog/pay/h5-open-mini`

### 5.7 图表与 Demo

前端页面：

- `client/packageB/pages/chart/chart.vue`：`qiun-data-charts`，当前为模拟数据渲染
- `client/packageB/pages/demo/demo.vue`：emoji 虚拟列表（`z-paging` 等 uni_modules）

### 5.8 管理后台（admin）业务模块

后台路由模块：

- `moment`：
  - `/moment/list`：动态列表（ProTable，展示头像/内容/图片/位置/点赞/浏览/时间）
  - `/moment/detail`：动态详情（当前为占位）
- `dashboard`：
  - `/welcome`：欢迎页
